# 普通人传记应用 PRD文档

## 1. 产品概述

### 1.1 创意名称
**"小传"** - 为每一个平凡人生写传记

### 1.2 创意介绍
传记不应是名人的专属，普通人也值得被记录。我们希望打造一个温暖的AI传记家，像微信好友一样陪伴老人聊天，在日常对话中自然地收集人生故事，最终整理成珍贵的传记，成为家族传承的精神财富。

### 1.3 产品愿景
让每一个普通人的故事都能被看见、被记住、被传承。

---

## 2. 目标用户及痛点

### 2.1 目标用户
- **核心用户**：50-80岁的老年人（希望记录自己的人生故事留给后代）
- **辅助用户**：子女（帮助父母记录、导出传记、创建纪念网站）

### 2.2 使用场景
1. **日常聊天**：像微信好友一样随时聊天，AI传记家在聊天中自然提问
2. **阶段整理**：用户设定周期后（如100天），自动整理传记初稿
3. **导出分享**：一键导出PDF/Word，或为父母创建个人纪念网站
4. **智能问答**：子女可以询问AI关于父母的问题/心愿

### 2.3 当前痛点
- 用户没有时间和能力系统整理自己的人生经历
- 普通人难以承担专业传记作者的费用
- 记忆会随时间流逝，重要故事容易被遗忘
- 老年人对复杂APP操作不适应

## 4. Agent人设与话术体系

### 4.1 Agent人设
- **姓名**：小传
- **形象**：温暖、耐心、专业的传记家
- **语气**：亲切、尊重、不催促
- **目标**：让老人感到被理解、被尊重，愿意分享自己的故事

### 4.2 开场话术
> "您好，我是小传，专门帮人记录人生故事的老朋友。您愿意和我聊聊您年轻时候的事吗？"

### 4.3 对话原则
1. **建立信任**：先闲聊，再深入提问
2. **循序渐进**：从轻松话题切入，逐步深入
3. **尊重意愿**：不强求，老人不想说的话题不追问
4. **积极反馈**：对老人的分享给予真诚的肯定和回应

## 5. 问题引导树

### 5.1 人生阶段划分
| 阶段 | 主题 | 目标 |
|------|------|------|
| 第一章 | 童年记忆 | 了解成长环境、家庭背景 |
| 第二章 | 求学经历 | 了解教育背景、人生转折点 |
| 第三章 | 青春岁月 | 了解年轻时候的梦想和追求 |
| 第四章 | 爱情家庭 | 了解婚姻、子女、家庭故事 |
| 第五章 | 事业奋斗 | 了解工作经历、职业成就 |
| 第六章 | 人生感悟 | 了解人生智慧、对后代的寄语 |

### 5.2 详细问题设计

#### 第一章：童年记忆（3-5个问题）
1. "您小时候生活在哪里？那个地方是什么样的？"
2. "您小时候家里有兄弟姐妹吗？你们关系怎么样？"
3. "您小时候最喜欢玩什么游戏？有没有特别难忘的童年趣事？"
4. "您印象最深的一位老师是谁？他/她对您有什么影响？"

#### 第二章：求学经历（3-5个问题）
1. "您上学的时候成绩怎么样？最喜欢哪门课？"
2. "有没有什么学习经历让您特别难忘？比如第一次离开家去外地读书？"
3. "您的第一份工作是什么？是怎么找到的？"

#### 第三章：青春岁月（3-5个问题）
1. "年轻时候有没有什么梦想？后来实现了吗？"
2. "您年轻时最骄傲的一件事是什么？"
3. "年轻时候有没有什么遗憾的事情？如果重来一次会怎么做？"

#### 第四章：爱情家庭（3-5个问题）
1. "您和老伴是怎么认识的？还记得第一次见面的场景吗？"
2. "结婚的时候是什么样的？有没有特别难忘的回忆？"
3. "有了孩子之后，您觉得最大的变化是什么？"
4. "您最想对子女说的一句话是什么？"

#### 第五章：事业奋斗（3-5个问题）
1. "您从事过最久的工作是什么？您喜欢这份工作吗？"
2. "工作中有没有遇到过特别大的困难？是怎么克服的？"
3. "您觉得自己在工作中取得的最大成就是什么？"

#### 第六章：人生感悟（3-5个问题）
1. "回首这一生，您觉得最重要的是什么？"
2. "您对年轻人有什么建议？"
3. "您最大的心愿是什么？"
4. "如果可以回到过去，您想回到哪个时刻？"

## 6. 用户体验设计

### 6.1 设计原则
- **老年人友好**：超大字体（默认20px+）、高对比度、温暖配色
- **简洁直观**：操作步骤尽量简化，避免复杂流程
- **温暖亲切**：使用暖色调（米色、浅橙色、淡红色），营造家的感觉
- **语音支持**：语音输入和语音朗读功能

### 6.2 配色方案
| 用途 | 颜色 | 色值 |
|------|------|------|
| 背景色 | 米色 | #FDF6E3 |
| 主色调 | 暖橙色 | #F59E0B |
| 辅助色 | 淡红色 | #EF4444 |
| 文字色 | 深灰色 | #374151 |
| Agent气泡 | 淡绿色 | #DCF8C6 |
| 用户气泡 | 白色 | #FFFFFF |

### 6.3 界面设计要点
1. **聊天界面**：模拟微信聊天样式，左侧Agent消息（绿色气泡），右侧用户消息（白色气泡）
2. **语音按钮**：每个消息右侧有语音朗读按钮，底部有语音输入按钮
3. **大字模式**：默认大字体，可切换超大字体模式
4. **操作提示**：清晰的操作指引，避免用户迷失

## 7. 技术方案

### 7.1 架构设计
- **前端**：单HTML文件，内嵌CSS和JavaScript，零依赖
- **AI能力**：预编排对话脚本 + 可选接入LLM API
- **数据存储**：localStorage本地存储

### 7.2 核心流程
```
注册登录 → 新手引导 → 开始聊天 → AI自然提问 → 
逐步收集故事 → 阶段整理 → 生成传记 → 导出分享
```

### 7.3 API接口设计（后续扩展）
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/chat | POST | AI对话接口 |
| /api/biography | GET | 获取传记 |
| /api/biography | POST | 生成/更新传记 |
| /api/user | POST | 用户注册 |
| /api/user/login | POST | 用户登录 |
