## 一、产品概述

### 1.1 产品定位

**小传**是一款面向中老年人群及其家人的AI人生传记记录应用，旨在通过温暖亲切的对话引导，帮助老人回忆并记录真实的人生故事，最终生成专业、完整、感人的长篇传记，作为家族传承的珍贵精神财富。

### 1.2 核心理念

- **真实记录**：所有内容必须基于用户真实讲述，绝对不能编造或虚构
- **温暖陪伴**：用AI陪伴老人聊天，像家人一样亲切关心
- **深度挖掘**：专业的访谈技巧，从细节到感受，深入挖掘人生故事
- **家族传承**：将个人经历转化为可传承的家族精神财富

### 1.3 目标用户

| 用户群体 | 描述 | 核心需求 |
|----------|------|----------|
| 中老年用户（60+） | 有丰富人生经历，希望记录自己的故事 | 方便记录、语音输入、温暖陪伴 |
| 子女/晚辈 | 希望为父母/祖辈记录人生故事 | 操作简单、内容真实、高质量传记 |
| 家庭用户 | 希望保存家族记忆和文化 | 家族传承、多人协作、长期保存 |

## 三、核心功能详述

### 3.1 目标设置（Onboarding）

**功能描述**：用户首次使用时设置记录目标，包括人物信息和传记偏好。

**流程**：
1. 输入人物称呼（如：爸爸、爷爷）
2. 选择关系类型（祖父母/父母/自己/其他）
3. 选择完成时间（7天/14天/30天）
4. 设置目标信件数量（3/5/10封）
5. 选择传记风格（温馨/正式/故事集）
6. 选择传记结构（时间线/重大事件）
7. 确认大纲并开始

**数据模型**：
```json
{
  "name": "称呼",
  "relation": "grandparent/parent/self/other",
  "targetTime": "7d/14d/30d",
  "letterCount": 5,
  "style": "warm/formal/story",
  "structure": "timeline/milestone",
  "createdAt": "ISO时间戳"
}
```

### 3.2 对话系统（Chat）

**功能描述**：核心对话界面，AI以温暖亲切的方式引导用户讲述人生故事。

**对话阶段**：

| 阶段 | 名称 | 目的 | 行为特征 |
|------|------|------|----------|
| trust_building | 破冰信任期 | 建立信任关系 | 纯粹闲聊，不涉及传记问题 |
| interest_exploration | 兴趣探索期 | 探索兴趣话题 | 轻松提及人生经历相关话题 |
| deep_collection | 深度采集期 | 正式信息采集 | 按传记阶段引导问题 |

**传记结构模式**：

**模式一：时间线模式**
- 童年记忆（1-18岁）
- 青春年华（18-28岁）
- 求学之路
- 初入职场
- 事业发展
- 爱情家庭
- 育儿之路
- 友情岁月
- 兴趣爱好
- 挑战与成长
- 难忘时刻
- 人生感悟

**模式二：重大事件模式**
- 人生第一次
- 挑战与成长
- 爱与关系
- 梦想与追求
- 人生智慧

**输入方式**：
- 语音输入（主要方式，适合老年用户）：绿色渐变按钮，白色"按住说话"文字，无图标，参考微信风格
- 文字输入（辅助方式）：底部输入框，左侧语音/文字切换按钮，右侧发送按钮

**记忆提取**：
- 对话结束后自动提取关键信息
- 提取内容包括：基础信息、偏好、关键记忆点
- 使用DeepSeek模型进行信息提取

### 3.3 信夹页面（Letterbox）

**功能描述**：展示已完成的信件和传记章节进度。

**功能点**：
- 信件列表展示（时间、标题、预览）
- 传记进度条（完成百分比）
- 剩余天数倒计时
- 章节列表
- 快速入口到传记阅读

### 3.4 传记生成（Biography Generation）

**功能描述**：基于用户记忆生成专业的长篇传记。

**传记结构**：
```
传记
├── intro（引言）：不少于800字，概括人物一生
├── ending（结语）：不少于1500字，总结人生感悟
├── epilogue（后记）：不少于500字，创作过程或读后感受
├── timeline（时间线）：25-30个关键时间节点
├── photos（照片位）：20个照片位，含标题和描述
├── chapters（章节）：15-20章，每章不少于2000字
│   ├── title（标题）
│   ├── year（时间范围）
│   ├── content（正文）
│   ├── quote（金句，25-40字）
│   └── emotion（情感标签）
└── familyView（家人视角）：不少于1000字
```

**写作要求**：
- 全文不少于30000字（三万字以上）
- 语言温暖亲切，适合中老年人阅读
- 所有内容必须真实，基于用户讲述
- 兼具文学性和艺术性，达到文学作品水准

**触发方式**：
- 用户在对话中明确提及"生成传记"、"查看传记"等
- 用户不满意当前传记时可重新生成

### 3.5 用户反馈（Feedback）

**功能描述**：收集用户反馈，支持通过"我的"页面或对话触发。

**反馈类型**：
- 功能建议 💡
- 问题反馈 🐛
- 不满意 😔
- 表扬 👍
- 其他 📝

**触发方式**：
1. "我的"页面点击「用户反馈」菜单
2. 对话中输入反馈相关关键词（反馈、意见、建议、问题等）

**上下文记录**：
- 默认包含完整对话上下文
- 上下文内容：聊天记录、用户记忆、信件数量、传记信息

**离线缓存**：
- 网络异常时保存到本地
- 网络恢复后自动上传

## 四、AI模型体系

### 4.1 模型配置

| 模型 | 用途 | 特性 |
|------|------|------|
| Minimax abab6-chat | 聊天对话 | 温暖亲切，适合日常对话和情感陪伴 |
| Minimax abab6-chat (Biography) | 传记创作 | 长篇叙事，文学写作，人生故事 |
| Minimax M3 | 兜底模型 | 全能模型，复杂推理，多模态 |
| Minimax vision-pro | 图片理解 | 多模态模型，支持图片分析和描述 |
| DeepSeek | 信息提取 | 逻辑推理，结构化输出，数据处理 |

### 4.2 模型降级策略

```
首选：云函数调用 Minimax abab6 → 失败
    ↓
备选：直接API调用 Minimax abab6 → 失败
    ↓
兜底：直接API调用 Minimax M3 → 失败
    ↓
报错：提示用户网络异常
```

### 4.3 传记评测体系

**评测维度**：
- 忠实度（Faithfulness）：内容是否基于用户真实讲述
- 丰富度（Richness）：内容是否充实丰富
- 文学性（Literary）：语言是否具有文学水准
- 情感表达（Emotion）：是否能引发情感共鸣
- 结构完整性（Structure）：章节结构是否完整合理

**评测流程**：
1. 使用LLM作为Judge对生成的传记进行评分
2. 记录各项评分和改进建议
3. 根据评测结果优化prompt和生成策略

## 七、用户体验设计

### 7.1 设计原则

- **适老化设计**：大字体、简洁界面、语音优先
- **温暖风格**：深色背景、金色点缀、玻璃拟态效果
- **流畅交互**：平滑动画、即时反馈、友好提示

### 7.2 视觉规范

| 元素 | 规范 |
|------|------|
| 主色调 | 深蓝色背景 (#0a0a1a → #1a1a2e) |
| 强调色 | 金色 (#d4a853) |
| 文字色 | 白色 (#ffffff) |
| 辅助色 | 浅灰色提示文字 |
| 字体大小 | 中号/大号字体，适合老年用户 |

### 7.3 交互规范

- 语音输入为主，文字输入为辅
- 操作按钮大小适中，方便点击
- 加载状态有明确提示
- 错误信息友好易懂
