## 二、测试流程

### 2.1 启动前检查

```
□ 确认代码已保存且无编译错误
□ 确认云开发环境已初始化
□ 确认网络连接正常
□ 确认开发者工具调试模式已开启
```

### 2.2 核心功能测试（必跑）

#### 模块1：登录与初始化

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 微信登录 | 成功获取用户信息，进入聊天页面 | 点击微信登录按钮 |
| 匿名登录 | 成功进入聊天页面，使用默认用户信息 | 点击匿名登录按钮 |
| 用户信息存储 | 用户信息正确保存到 localStorage | 检查 wx.getStorageSync('userInfo') |
| 目标设置 | 设置记录对象后正确保存 | 进入"我的目标"页面设置 |
| 首次引导 | 首次登录显示交互式教程 | 清除缓存后重新登录 |

#### 模块2：聊天功能

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 文字输入 | 输入文字后发送成功 | 点击输入框输入文字 |
| 语音输入 | 语音转文字成功 | 按住麦克风按钮说话 |
| 语音/文字模式切换 | 模式切换正常，图标正确显示 | 点击切换按钮 |
| 消息发送 | 消息显示在聊天列表 | 发送文字/语音消息 |
| 消息接收 | AI回复正常显示 | 等待AI响应 |
| 网络异常处理 | 显示错误提示，自动重试 | 断开网络后发送消息 |
| 聊天记录保存 | 聊天记录持久化存储 | 重启小程序后检查记录 |
| 传记意图检测 | 输入"传记"触发传记生成 | 在聊天框输入"生成传记" |

#### 模块3：信夹页面

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 章节列表显示 | 显示所有人生章节 | 进入信夹页面 |
| 章节状态更新 | 完成对话后状态变为"已完成" | 完成一个章节的对话 |
| 信件保存 | 章节完成后信件正确保存 | 完成对话后检查 |
| 进度显示 | 顶部进度条正确显示 | 查看进度条 |

#### 模块4：传记生成

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 传记生成 | 生成完整传记（≥10000字） | 完成所有章节或触发传记生成 |
| 传记显示 | 在"查看传记"页面正确显示 | 点击"我的"→"查看传记" |
| 传记章节 | 包含引言、章节、结语、家人视角 | 检查传记结构 |
| 重新生成 | 根据用户反馈重新生成 | 输入修改意见 |
| 传记存储 | 传记持久化存储 | 重启小程序后检查 |

#### 模块5：用户引导教程

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 教程启动 | 首次登录显示教程覆盖层 | 清除缓存后重新登录 |
| 高亮定位 | 高亮区域准确定位到目标元素 | 检查每一步的高亮位置 |
| 步骤导航 | 点击下一步/跳过正常切换 | 点击按钮 |
| 教程完成 | 完成后不再显示 | 完成教程后重新进入 |

#### 模块6：错误处理

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 网络错误 | 显示友好错误提示 | 断开网络 |
| LLM调用失败 | 自动降级到备用模型 | 模拟API调用失败 |
| 空数据处理 | 显示默认内容或提示 | 清除所有缓存 |
| 页面异常 | 页面正常加载，不崩溃 | 快速切换页面 |

### 2.3 埋点与监控测试（必跑）

#### 模块7：埋点追踪

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 页面访问埋点 | page_view事件正常上报 | 打开任意页面 |
| 点击埋点 | click事件正常上报 | 点击按钮 |
| API调用埋点 | api_call事件正常上报 | 触发网络请求 |
| LLM调用埋点 | llm_call事件正常上报 | 发送聊天消息 |
| 错误埋点 | error事件正常上报 | 触发错误场景 |
| 性能埋点 | performance事件正常上报 | 监控页面加载时间 |

#### 模块8：错误监控

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 运行时错误捕获 | wx.onError捕获并上报 | 触发JS错误 |
| Promise拒绝捕获 | wx.onUnhandledRejection捕获 | 触发未处理的Promise拒绝 |
| 网络错误捕获 | network_error事件上报 | 模拟网络请求失败 |

#### 模块9：性能监控

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 页面加载时间 | page_load_time事件上报 | 打开页面 |
| API响应时间 | api_call事件包含duration | 触发API调用 |
| LLM响应时间 | llm_call事件包含duration | 发送聊天消息 |

---

### 2.4 传记评测测试（可选）

#### 模块10：评测体系

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 评测运行 | 成功运行评测样本 | 在测试页面点击"开始评测" |
| LLM评分 | AI给出合理评分（60-100分） | 查看评测结果 |
| 迭代优化 | 多轮迭代后评分提升 | 点击"迭代优化" |
| 评测存储 | 评测结果保存到storage | 检查 biographyEvaluations |

### 2.5 兼容性测试（必跑）

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| iOS兼容性 | 页面正常显示，功能正常 | 在iOS真机测试 |
| Android兼容性 | 页面正常显示，功能正常 | 在Android真机测试 |
| 不同屏幕尺寸 | 布局自适应 | 在不同尺寸模拟器测试 |
| 横竖屏切换 | 页面正常布局 | 切换横竖屏 |

### 2.6 性能测试（可选）

| 测试项 | 预期结果 | 测试方法 |
|--------|---------|---------|
| 启动时间 | < 3秒 | 冷启动小程序 |
| 页面切换 | < 1秒 | 切换页面 |
| 内存使用 | < 500MB | 使用开发者工具监控 |
| CPU占用 | < 50% | 使用开发者工具监控 |

## 五、测试执行清单

每次代码更新后，按以下顺序执行测试：

```
阶段1: 编译检查
  □ 检查是否有编译错误
  □ 检查是否有警告

阶段2: 核心功能测试
  □ 登录与初始化
  □ 聊天功能
  □ 信夹页面
  □ 传记生成

阶段3: 用户体验测试
  □ 用户引导教程
  □ 页面布局
  □ 交互流畅度

阶段4: 错误处理测试
  □ 网络错误
  □ 空数据处理
  □ 异常场景

阶段5: 埋点与监控测试
  □ 埋点事件上报
  □ 错误监控
  □ 性能监控

阶段6: 兼容性测试
  □ iOS真机
  □ Android真机

阶段7: 生成测试报告
  □ 记录问题
  □ 评估影响
  □ 提出改进建议
```
