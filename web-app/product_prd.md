## 1. 产品概述

### 1.1 产品定位
小传是一款AI驱动的个人传记生成应用，通过与用户对话采集人生故事，最终生成完整的传记作品。用户可以将人生故事讲给孩子听，留下珍贵的记忆遗产。

### 1.2 核心理念
- **真实记录**：基于真实对话内容，不虚构、不编造
- **温暖陪伴**：AI作为对话伙伴，耐心倾听、深度交流
- **循序渐进**：分阶段引导用户回忆，不急于求成
- **珍贵遗产**：生成的传记是留给后代的珍贵礼物

### 1.3 目标用户
- 希望记录人生故事的中老年人
- 希望为长辈记录人生经历的年轻人
- 对个人传记写作感兴趣的所有用户

### 1.4 与小程序版的关系
网页版与微信小程序版保持功能和体验的一致性，同时针对Web平台特性进行优化：
- 共享设计语言和品牌风格
- 核心功能完全一致（对话、信夹、传记、反馈）
- 网页版支持更多平台访问（PC浏览器、移动端浏览器）

## 3. 核心功能详述

### 3.1 登录模块

**页面：** LoginPage

**功能说明：**
- 用户通过输入昵称登录
- 支持上传自定义头像
- 需要勾选用户协议才能登录
- 提供游客模式，无需登录即可体验

**界面设计：**
- 深色星空背景，营造温馨氛围
- 居中布局，品牌logo突出
- 微信绿色登录按钮，与小程序保持一致
- 底部显示用户协议链接

### 3.2 目标设置模块

**页面：** OnboardingPage

**功能说明：**
- 设置记录对象（写给谁）
- 选择传记结构（时间线/里程碑）
- 选择叙事风格（温暖/正式/故事）
- 设置目标信件数量和完成时间

### 3.3 对话模块

**页面：** ChatPage

**功能说明：**
- 与AI进行自然对话
- 语音输入（Web Speech API）：绿色渐变按钮，白色"按住说话"文字，无图标，参考微信风格
- 文字输入，支持Enter发送：底部输入框，左侧语音/文字切换按钮，右侧发送按钮
- 对话阶段展示（破冰信任期→兴趣探索期→深度采集期）
- 传记进度展示（百分比、信件数、剩余天数）
- 话题建议卡片，点击直接发送
- 消息朗读功能
- 字体大小调节
- 反馈意图检测（输入关键词自动触发反馈流程）

**界面设计（贴近小程序版）：**
- 浅色渐变背景（#eef2f7 → #f5f7fa）
- 白色卡片式消息气泡
- AI消息：白色背景，圆角左上角较小
- 用户消息：绿色渐变背景（#07c160 → #06ad56），圆角右上角较小
- 金色主题色（#D4A853）用于强调元素
- 阶段指示器使用金色圆点和线条
- 进度条使用金色到红色渐变

### 3.4 信夹模块

**页面：** LetterboxPage

**功能说明：**
- 展示所有已写的信件
- 未读信件有红点标记
- 点击信件查看详情
- 支持标记已读

### 3.5 传记模块

**页面：** BiographyPage

**功能说明：**
- 展示生成的完整传记
- 章节导航
- 支持重新生成

### 3.6 我的模块

**页面：** MyPage

**功能说明：**
- 用户信息卡片（头像、昵称、登录方式、在线状态）
- 统计数据（已写信件、记录对象、陪伴天数）
- 功能菜单（我的目标、查看传记、示例传记、关于我们、用户反馈、清除记忆、退出登录）
- 底部版权信息

**界面设计（贴近小程序版）：**
- 深色背景（#0a0a1a → #1a1a2e）
- 星空粒子动画效果
- 毛玻璃卡片效果
- 金色主题色强调数据

### 3.7 用户反馈模块

**页面：** FeedbackPage

**功能说明：**
- 反馈类型选择（功能建议、问题反馈、投诉、表扬、其他）
- 反馈内容输入（最多2000字）
- 联系方式输入（选填）
- 上下文包含开关（用户可选择是否包含完整对话上下文）
- 上下文自动压缩（超过4000 tokens时自动截断）
- 本地缓存机制（网络异常时保存到localStorage）
- 自动重试上传（应用启动时检测并上传缓存的反馈）

**触发方式：**
1. 通过"我的"页面进入
2. 通过对话关键词触发（反馈、意见、建议、投诉、问题、bug等）

## 4. AI模型体系

### 4.1 模型配置

| 模型名称 | 供应商 | 用途 |
|----------|--------|------|
| Minimax abab6 | Minimax | 日常对话、记忆提取 |
| Minimax M3 | Minimax | 传记生成（高级模型） |
| DeepSeek | DeepSeek | 降级备用、特定任务 |

### 4.2 模型降级策略
- 优先使用Minimax abab6进行对话
- 传记生成使用Minimax M3
- 当主模型不可用时，自动切换到DeepSeek

### 4.3 评测体系（未来规划）
- 建立传记生成评测集（多轮对话+最终传记）
- 使用LLM as a Judge进行自动评分
- 持续迭代优化传记生成质量

## 5. 数据存储与管理

### 5.1 本地存储

| 存储键 | 内容 | 说明 |
|--------|------|------|
| `currentUser` | 用户信息 | 登录状态 |
| `messages` | 消息列表 | 当前对话 |
| `letters` | 信件列表 | 已写的信 |
| `goal` | 目标设置 | 传记目标 |
| `userMemory` | 用户记忆 | AI记住的信息 |
| `notifications` | 通知列表 | 系统通知 |
| `localFeedbacks` | 本地反馈缓存 | 离线时的反馈 |

### 5.2 云端数据库（小程序版使用）
网页版暂不使用云端数据库，所有数据存储在本地。

## 6. 用户体验设计

### 6.1 设计原则
- **一致性**：与小程序版保持视觉和交互一致
- **简洁性**：减少视觉干扰，突出内容
- **温暖感**：金色主题传递温暖和珍贵感
- **响应式**：适配各种屏幕尺寸

### 6.2 视觉规范

| 元素 | 颜色 | 值 |
|------|------|------|
| 主背景 | 深色渐变 | #0a0a1a → #1a1a2e（我的页面） |
| 主背景 | 浅色渐变 | #eef2f7 → #f5f7fa（对话页面） |
| 主题色 | 金色 | #D4A853 |
| 主题色暗 | 深金色 | #b89444 |
| 成功色 | 绿色 | #07c160 |
| 危险色 | 红色 | #e74c3c |
| 文字 | 白色 | #ffffff |
| 文字次要 | 灰色 | rgba(255, 255, 255, 0.5) |

### 6.3 交互规范
- 按钮点击有缩放反馈（scale 0.98）
- 输入框聚焦有边框高亮
- 页面切换有淡入动画
- 加载状态有动画反馈
